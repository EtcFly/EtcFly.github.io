2. 通讯方式

2.1. 管道

  在unix中, 管道属于较早的ipc通信方式之一, 管道分为两种, 有名管道 和 无名管道, 当中无名管道只能用于存在亲缘关系的进程。管道有如下通用特性:

  • 半双工通信(某些平台可能实现为全双工, 验证linux 4.15属于半双工)， fd0数据读, fd1用于数据写
  • 需要进程存在亲缘关系(有名管道不存在该限制)

其次, 对于管道存在如下性质:

  • 当写端打开情况下:
    • 如果管道无数据, 读端读取数据将阻塞
    • 如果管道有数据, 读端将正常读取数据
  • 当写端关闭情况下:
    • 如果管道无数据, 读端返回0
    • 如果管道有数据, 读端将正常读取数据
  • 当读端打开情况下:
    • 如果管道满, 则写端写入将阻塞
    • 如果管道非满, 则写端正常写入数据
  • 当读端关闭情况下:
    • 写端写入数据将直接出发SIGPIPE信号异常。

2.1.1.1 相关API
pipe

原型: int pipe(int pipefd[2])
参数:

  • pipefd[2]: 打开的管道
    • pipefd[0]: 读管道
    • pipefd[1]: 写管道

返回: 0: 成功 -1: 失败

示例:

pipe2

原型: int pipe2(int pipefd[2], int flags)
参数:

  • pipefd[2]: 打开的管道
    • pipefd[0]: 读管道
    • pipefd[1]: 写管道
  • flags: 需要为管道设置的文件标志
    • O_CLOEXEC: 同Open调用的标志
    • O_DIRECT: 发送的数据均是一个个独立数据包, 当长度大于PIPE_BUF将是多个数据包, 如果一次读取缓存小于要读取的数据包内容, 将读取缓存的大小的数据, 超过的被丢弃。同时, 0长度的数据包是不支持的。
    • O_NONBLOCK: 设置管道是否阻塞。

返回: 0: 成功 -1: 失败

示例:

popen

原型: FILE *popen(const char *command, const char *type)
参数:

  • command: 需要执行的命令
  • type: "r": 读类型 "w": 写类型

返回: 成功后返回管道句柄 否则为NULL

2.1.2. 命名管道

  命名管道和无名管道的重要区别就是它不强制管道两端进程存在亲缘关系, 命名管道会在文件系统中创建一个管道文件, 其他进程可以通过打开该管道文件来与其通信。但是需要注意如下几点:

  • 创建管道的文件的目录不应该存在同管道文件相同文件名的文件, 否则创建报错
  • 管道创建后, 管道读打开需要阻塞到其他进程对管道进行首次写打开为止。管道写打开需要阻塞到其他进程对管道进行手册读打开为止。
  • 单次操作需要小于PIPE_BUF, 否则服务保证写入管道数据的原子性。

2.1.2.1 相关API
mkfifo

原型: int mkfifo(const char *pathname, mode_t mode)
参数:

  • pathname: 需要创建的管道名
  • mode: 创建的管道权限(如0644)

返回: 0: 成功 -1:失败

示例:

mkfifoat

原型: int mkfifoat(int dirfd, const char *pathname, mode_t mode)
参数:

  • dirfd:
    • 如果pathname为绝对目录, 则忽略该字段, mkfifo和mkfifoat功能类似。
    • 如果pathname为相对目录。
      • 如果dirfd值为AT_FDCWD, 那么mkfifo和mkfifoat功能类似。
      • 如果dirfd值是目录句柄(使用open并指定O_DIRECTORY), 那么pathname为相对于该目录句柄的fifo管道。
  • pathname: 管道文件名
  • mode: 打开的模式(如0644)

返回: 0: 成功 -1:失败
