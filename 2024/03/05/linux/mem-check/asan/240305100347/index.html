使用asan进行内存排查

本文最后更新于：2024年11月4日 下午

本文作者：EtcFly
原文地址：https://etcfly.top

简介

  asan是一款由google开发的内存错误检测工具, 最早在LLVM V3.1中引入, 由于其良好的检测性能, gcc从4.8版本开始也引入了AddressSanitizer支持，以支持内存相关的消杀，目前你只要在代码编译时加入特定的gcc选项就可以使用它。

  目前支持检测的内存问题类型有:

  • Use after free
  • Heap buffer overflow
  • Stack buffer overflow
  • Global buffer overflow
  • Use after return
  • Use after scope
  • Initialization order bugs
  • Memory leaks

相比于valgrind之类的工具(慢10-50倍), asan显得非常快速(仅减慢约2倍), 具体 可参见。因此在项目开发阶段, 很多时候都会首先使用asan来进行一个内存消杀, 已发现未知的问题。当然, 任何事物有优点也有缺点, asan也有不足:

  • 某些场景内存问题无法发现
  • 使用时必须添加编译参数并重新编译构建项目代码
  • asan本身会产生较大内存开销

原理分析

  工具主要由两部分组成:

  • 运行时库(用于动态内存分析):

运行时库（libasan.so.x）会接管malloc和free函数。malloc执行完后，已分配内存的前后（称为“红区”）会被标记为“中毒”状态，而释放的内存则会被隔离起来（暂时不会分配出去）且也会被标记为“中毒”状态。

  • 编译器编译插桩模块

如果使用了对应的编译选项, 编译器在编译阶段, 会对代码中每次的内存访问前添加检查代码, 用于检测静态相关内存访问的合法性

影子内存映射
  上面描述的 “红区”, 其实就是在分配内存前后多分配的空间, 这些空间在分配前被填充为特定值并被映射到影子内存中, 因此, 如果在一次内存访问的时候检测到红区异常, 那么可发现此次内存出现踩踏问题。

  asan将虚拟地址空间分为两个不相交的内存:

  • 主应用程序内存：该内存由常规应用程序代码使用
  • 影子内存(shadow): 该内存包含影子值, 影子内存和主应用程序之间存在对应关系。

主内存中1个字节映射为影子内存1bit, AddressSanitizer将 8 字节的应用程序内存映射到 1 字节的影子内存。对于任何对齐的 8 个字节的应用程序内存，只有 9 个不同的值：

  • qword 中的所有 8 个字节均未中毒（即可寻址）。影子值为0。
  • qword 中的所有 8 个字节都已中毒（即不可寻址）。阴影值为负
  • 第一个k字节未中毒，其余字节8-k已中毒。影子值为k。

在访问主应用内存时, 一般先检查影子内存, 编译器执行检测如下:

MemToShadow(ShadowAddr)落入ShadowGap不可寻址的区域。因此，如果程序尝试直接访问影子区域中的内存位置，它将崩溃

具体在64bits和32bits机器上影子内存和主存对应如下:

安装

  从gcc4.8开始, asan被加入到gcc, 但不支持符号信息，无法显示出问题的函数和行数。从 4.9 开始，gcc 支持 AddressSanitizer 的所有功能。因此gcc 4.8以上版本使用ASAN时不需要安装第三方库，通过在编译时指定编译CFLAGS即可打开开关。

使用配置

  asan有很多检测选项, 不需要的选项可能会导致程序性能的下降， 因此需要按需开启，配置主要有两种方式:

  • 通过编译时给gcc传递编译参数
  • 通过ASAN_OPTIONS环境变量配置
  1. 编译参数
    主要选项有:
  • -fsanitize=address: 默认开启内存检测
  • -fsanitize-recover=address: 检测到内存错误不退出进程, 默认退出
  • -fno-stack-protector: 禁止栈保护
  • -fno-omit-frame-pointer: 按照经典arm栈帧格式压栈
  • -fsanitize-address-use-after-scope: 检测变量超出范围
  1. ASAN_OPTIONS设置：

ASAN_OPTIONS是Address-Sanitizier的运行选项环境变量。
主要有:

  • halt_on_error: 检测内存错误后继续运行
  • detect_leaks=1: 使能内存泄露检测
  • malloc_context_size=15: 内存错误发生时，显示的调用栈层数为15
  • log_path=/home/asan.log: 内存检查问题日志存放文件路径

设置方式有两种:

  • 环境变量导出
  • 命令执行前引入
