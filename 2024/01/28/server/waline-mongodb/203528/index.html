服务器应用迁移总结

本文最后更新于：2024年3月5日 凌晨

版权归作者所有, 转载请保留该部分声明。
本文作者：EtcFly
原文地址：https://etcfly.top

简介

  由于购买的服务器到期, 原先部署在该台服务器上的数据需要导出并重新部署到另一台ECS服务器上, 加之原先的服务全部部署在宿主机上, 应用之间依赖关系复杂, 因此这里打算全部使用容器化部署, 减少后期服务迁移的成本, 本次迁移过程中也走了一些弯路。因此这里记录一下, 避免以后重复出错。

首先说说本次需要迁移的应用和数据:

  • Mysql数据库
  • Mongodb数据库
  • Nginx代理服务
  • Umami流量检测
  • Waline评论系统
  • hexo 博客系统

  另外, 由于umami官方版本从v1.39.5更新到v2.9.0, 数据存储格式也发生了变化, 因此本次迁移还涉及到数据库兼容性转换, 接下来将一一介绍。
