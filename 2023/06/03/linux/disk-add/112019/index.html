ubuntu新硬盘挂载

简介

  ubuntu 有时候磁盘空间不足, 需要进行扩容, 这里对新硬盘挂载的流程进行一个介绍。通常一个新硬盘要想挂载, 需要有几个步骤:

  • 硬件连接
  • 分区写入
  • 格式化分区
  • 文件系统挂载

这里对这几个步骤进行详细的描述。

硬件连接

  硬盘要想连接到电脑, 需要通过STAT接口或者其他硬盘接口连接起来, 这是磁盘挂载的基础。

分区写入

格式化分区

  当硬盘分区划好以后, 还需要进行文件系统的格式化, 不然挂载的时候是会报错。这里格式化为ext4文件系统。

永久挂载

永久挂载需要编辑/etc/fstab文件。

  • file system: 需要挂载的磁盘分区
  • mount point: 挂载点
  • type: 挂载的文件系统类型(如ext2、ext3、ext4)
  • option: 指定挂载选项, 默认为defaults, 也可以换其他选项
选项备注
Async/sync设置是否为同步方式运行, 默认为async
auto/noauto当运行mount -a命令时, 此文件系统是否被主动挂载。默认为auto
rw/ro是否以只读或者读写模式挂载
exec/noexec限制此文件系统内是否能够进行 “执行” 操作
user/nouser是否允许用户使用mount命令挂载
suid/nosuid是否允许suid的存在
usrquota启动文件系统支持磁盘配额模式
grpquota启动文件系统对群组磁盘配额模式的支持
defaults同时具有rw,suid,dev,exec,auto,nouser,async等默认参数的设置
  • dump: 表示挂载后的文件系统是否能被dump备份
    • 0 表示不能
    • 1 表示每天都进行dump备份
    • 2 表示不定期进行dump操作。
  • pass: 表示开机过程中是否校验扇区
    • 0 表示不要校验
    • 1 表示优先校验（一般为根目录）
    • 2 表示为在 1 级别校验完后再进行校验；

测速

  • 测试硬盘读速度: hdparm

    • 测试buffer缓存下硬盘的读取速度。
    • 测试cache缓存下硬盘读取速度。
    • 测试不用buffer缓存情况下，读取10g数据的速度，默认一个offset=1G速度。

  • 测试硬盘读写速度:dd
    dd用于复制，从if读出，写到of，测试出的结果均为顺序读写速度, if=/dev/zero不产生IO，因此可以用来测试纯写速度。同理of=/dev/null不产生IO，可以用来测试纯读速度, bs是每次读或写的大小，即一个块的大小，count是读写块的数量。

    • 测试纯写速度，8k 的块大小写 125000 次
    • 测试纯读速度，8k的块大小写125000次

    • 上述dd命令完成前并没有让系统真正把文件写到硬盘上，dd先把数据写到操作系统“写缓存”，就完成了写操作。需要调用sync函数，把“写缓存”中的数据刷入硬盘。

      • -fsync 是将数据已经写入硬盘，但是是在经过缓存后最后再写入硬盘
      • -dsync 可以当成是模拟数据库插入操作，在/dev/zone中读出一条数据就立即写入硬盘，速度会很慢。

问题

  • VFS:can‘t find ext4 filesystem: 分区格式化有问题, 可以重新使用mkfs -t ext4 /dev/sdb重新格式化。

linux挂载磁盘后启动时报错 VFS:can‘t find ext4 filesystem

  • 系统启动时报Failed to connect to lvmetad. Falling back to device scanning

可以通过journalctl -xe 查看具体开机错误细节并针对细节排查。
