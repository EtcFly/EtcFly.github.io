Linux下Stm32开发环境搭建

本文最后更新于：2024年3月5日 凌晨

版权归作者所有, 转载请保留该部分声明。
本文作者：EtcFly
原文地址：https://etcfly.top

背景

  Windows下进行stm32开发, 通常有Keil, IAR之类的集成开发环境, 这些环境集成了代码编辑, 在线调试, 固件烧录等功能, 大大方便了开发者, 但是当项目变得复杂庞大, 传统的IDE工具很难完成相关工作。本文着重在Linux 环境下进行Stm32开发, 通过Makefile配合Jlink完成整个开发过程。

实验环境:

  • 硬件: ARM Cortex-M3核心STM32f103zet6
  • gdb: GNU gdb (GNU Arm Embedded Toolchain 10.3-2021.10) 10.2.90.20210621-git
  • jlink: SEGGER J-Link Commander V7.84f (Compiled Feb 7 2023 16:51:58)

环境准备

  • 交叉编译环境
  • JLink固件烧录和在线调试
  • Make 相关工具

交叉编译环境

  stm32的交叉编译环境是arm-gcc-eabi-none, eabi表示Embedded Application Binary Interface, none表示无操作系统。

  • 下载链接

由于这里下载是tar包, 因此只需要解压并将可执行文件目录添加到系统环境变量即可.
