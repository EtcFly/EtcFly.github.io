基于公有云+frp搭建内网穿透服务

简介

  有时候需要通过外网访问内网的相关资源，但是由于内网主机往往没有公网的IP只能通过NAT服务访问外网, 这导致外网无法定位内网主机位置。但是传统的ddns很难穿透路由器防火墙，因此只能通过花生壳这类软件进行域名ip动态映射。由于花生壳这类软件是收费得，免费版本具有流量限制，因此使用具有较大限制。
  本文将基于公有云服务器 + 内网主机搭建内网穿透服务，来替换收费版本的花生壳服务。原理在于本地主机一直与远端服务保持TCP长连接，以此保证外网可以通过tcp连接访问内网资源。

问题

  • frp start error: port unavailable
    这个一般是客户端remote_port和server_port重叠了, 需要修改成不同的。因为frp程序需要监控两个端口。
  • [E] [service.go:273] token in login doesn't match token from configuration
    这个一般是客户端和服务端的Token不一致导致的, 修改二者的Token并使其保持一致。
