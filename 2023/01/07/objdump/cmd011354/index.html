objdump参数

简介

  objdump是 unix 下显示目标文件信息的一个GNU工具。objdump 显示有关一个或多个目标文件的信息(包括.a、.elf、.hex等格式)。这些选项控制着需要显示的特定信息。这些信息对从事编译工具工作的程序很有用, 而不是那些只想要他们的程序通过编译并且运行工作的程序员。

格式

objdump [OPTIONS] OBJFILES

参数

  • -a | --archive-header: 显示特定目标文件的信息。主要是显示文件相关格式信息。
    如:

  • -b bfdname | --target= bfdname: 指定目标文件的目标代码格式是 bfdname。这个选项可能不是必需的, objdump可以自动识别多种格式。
    如
    objdump -b oasys -m vax -h fu.o

  • -C | --demangle[=style]: 获取符号信息

  • -g | --debugging: 获取debug信息

  • -d | --disassemble: 反汇编 objfile 中机器指令汇编助记符, 该选项仅反汇编指令部分, 数据部分并不进行反汇编并展示。
    注意
  • 有时候返回时需要指定系统的架构, 如 objdump -d -m arm file.elf, -m arm 指定系统架构为arm架构。 -d指示反汇编指令部分。
  • -D | --disassemble-all: 反汇编所有部分, 包括数据部分。
    如:
  • objdump -D -m arm file.elf, 反汇编所有固件部分。
  • -h | --section-headers | --headers: 显示固件各 段信息表。可以用这个来确认系统启动时, .data数据段拷贝是否正确。
    如:
  • -i | --info: 使用 -b 或 -m 显示一个列表，其中显示规范可用的所有体系结构和对象格式。
  • -S | --source: 反汇编时同时显示源代码
  • --start-address=address | --stop-address=address: 设置其实和结束地址(通常配合一些其他选项一起使用)
    • -d: 反汇编起始结束地址
    • -r: 显示重定向入口
    • -R: 显示动态库的重定向入口

如:

  • -t | --syns: 显示文件的符号信息, 类似于nm -s main.elf

显示格式:

符号属性:

  • l : 本地符号(local)
  • g : 全局符号(global)
  • u : 全局唯一*(unique global)
  • ! : 即使全局也是局部
  • w : 符号是weak 或者是space
  • C : 符号是一个构造函数或者space
  • W : 该符号是一个警告或space
  • I : 符号是一种间接引用到另一个符号
  • i :
  • d : 该符号是一个调试符号
  • D : 该符号是一个动态符号或者space
  • F : 符号是一个函数的名称
  • f : 该符号是一个文件名
  • O ： 该符号是一个对象或者一个正常的符号(space)
  • -T : 和-t类似, 显示动态文件的符号信息
  • -x | --all-headers: 显示所有的符号信息, 相当于同时使用-a -f -h -p -r -t
