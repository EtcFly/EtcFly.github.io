搭建内网穿透服务

简介

  内网穿透具有很多的优势, 比如你想在公司访问家里私人电脑, 又比如你在家里搭建一台linux服务器对外提供访问, 正常情况下由于服务提供商的NAT转换, 我们没法具有一个有效的公网IP, 这时候是无法访问内网资源的，就可以通过内网穿透，访问原本无法访问的内网资源。
  内网穿透手段其实很多, 比如通过

  • 路由器ddns进行穿透,
  • 通过第三方软件服务商(花生壳或者其他的)
  1. 通过路由器ddns进行穿透比较方便, 只需要路由上配置即可, 但是得保证电信运营商的网关(如电信的天翼网关)上允许该服务。一般在家庭的入网第一个网关或者路由上进行配置，这比较麻烦。
  2. 通过第三方服务商进行穿透。需要安装第三方软件，以及相关账号，这里以花生壳为例，进行讲解。

SSH登陆

  一般我们拿到的结果是 tcp://XXX:port, 这里xxx就是域名, 用于SSH登陆的主机名, port表示端口号, 此时系统就会自动访问花生壳官方DDNS服务, 由其服务程序访问我们本地的花生壳客户端, 通过之前配置的映射表， 本地客户端会将其数据映射到我们主机的127.0.0.1:22端口啦！由此开启了快乐的内网穿透之旅。

远端挂载本地磁盘

  通过以上的步骤我们已经可以通过 ssh 连接本地主机了, 有的时候在远端登录主机时需要在两台主机中传输文件, 那么这时候就可以将远端主机通过 sshfs 挂载到本地。

安装sshfs
  • 执行

sudo apt install sshfs

挂载

sshfs [loginName@]host:remoteDir localMountDir [-o -p port]
如:
sshfs etcfly@6284qu5167.zicp.fun:/home/etcfly /home/etcfly/Documents/home-pc -o -p 29210

这时候磁盘就已经挂载成功了! 但是如果系统重启需要重新挂载, 如果需要永久挂载, 那么需要配置 /etc/fstab 文件添加如下:

  • 无密码

sshfs#[accountName@]x.x.x.x:[remoteDir] [localMountDir] fuse.sshfs defaults 0 0

  • 有密码

sshfs#[accountName@]x.x.x.x:[remoteDir] [localMountDir] fuse.sshfs IdentityFile=~/.ssh/id_rsa defaults 0 0
