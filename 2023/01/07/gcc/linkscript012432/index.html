gcc链接脚本语法

简介

  命令语言(Command Language)提供对链接过程的明确控制, 允许完整指定链接器的输入文件与其输出之间的映射。它控制：

  • 输入文件
  • 文件格式
  • 输出文件布局
  • 节地址(addresses of sections)
  • 公共块的放置

  您可以通过-T选项显式地提供命令文件(也称为链接描述文件), 隐式链接器脚本仅应在您想要扩充而不是替换默认链接器脚本时使用。 通常，隐式链接描述文件仅包含 INPUT 或 GROUP 命令。

  如果链接器打开的文件无法识别为受支持的对象或存档格式，也无法识别为链接描述文件，则会报告错误。

secname: 块名(gcc编译器编译出的程序具有多种块名,如
    .text块: 代码块,所有生成的代码都是以代码块存在。
    .data块: 数据块, 用于存放代码中一些可修改的全局变量的初始值, 初始值为0的不在范围内。
    .rodata: 用于存放代码中一些不可修改的值, 如const等修饰的值。
    .bss: 用于代码中初始值为0或者没有初始化的全局变量的存放
    .COMMON: 一般用于存放没有初始化的全局变量,但是有的时候编译器会直接放到bss段,这取决于编译器的行为 )

start: 代码运行地址
BLCOK(align): 一般是BLOCK(4)表示块4字节对齐
(NOLOAD): 表示该段不加载
AT(ldadr): 指定程序的加载地址(及程序存储的位置)
contents: 一般为.text, .data, .rodata, .bss, .COMMON
    如{ *(.bss) 表示所有的程序的bss段 } 还可以单独指定一个程序中的各个段到不同地方

基本的链接脚本规范

脚本的算术运算

GCC链接脚本支持基本的加减乘除,&以及赋值操作。许多有用的命令都涉及算术表达式。命令语言中表达式的语法与 C 表达式的语法相同，具有以下特点：

  • 所有表达式都被评估为整数并且是long或 unsigned long 类型。
  • 所有常量都是整数。
  • 提供了所有 C 算术运算符。
  • 可以引用、定义和创建全局变量。
  • 可以调用特殊用途的内置函数。

整数

  1. 八进制数表示(0后跟零个或多个八进制数字（01234567))
  1. 十进制整数以非零数字开头，后跟零个或多个数字（0123456789)
  1. 一个十六进制整数是0x要么0X后跟一个或多个从中选择的十六进制数字0123456789abcdefABCDEF
  1. 要写入负整数，请使用前缀运算符-
  1. 后缀K和M可用于分别缩放常量

符号名称

除非引用，否则符号名称以字母、下划线或点开头，并且可以包括任何字母、下划线、数字、点和连字符。未加引号的符号名称不得与任何关键字冲突。您可以通过用双引号将符号名称括起来来指定包含奇数字符或与关键字同名的符号：

由于符号可以包含许多非字母字符，因此用空格分隔符号是最安全的。 例如，A-B是一个符号，而A - B 是一个涉及减法的表达式。

位置计数器

特殊链接器变量点 ., 始终包含当前输出位置。由于.始终指的是输出部分中的位置，因此它必须始终出现在SECTIONS命令的表达式中。该.符号可以出现在表达式中允许使用普通符号的任何地方，但它的赋值有副作用。为符号赋值.将导致移动位置计数器。 这可用于在输出部分创建空白空洞。位置计数器可能永远不会向后移动。
