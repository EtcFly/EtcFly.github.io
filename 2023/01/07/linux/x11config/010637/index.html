ubuntu18.04 x11搭建远端服务界面

简介

  使用windowns+linux开发环境的时候, 经常需要交互式操作, 但是通过服务器ssh连接或者第三方虚拟机搭建的开发环境, 由于两个系统之间的割裂, 使得交互式体验不是很好, 这时候可以使用x windown技术很好的解决此类问题。通过x windown技术, 远端服务器充当x client, 本地以x server运行, 当远端服务器需要图像绘制, 将通过x client把绘制信息通过x windowns协议递交给x server服务端, 由服务端进行图形绘制。缩小系统的割裂。

准备

  • 本地ssh + 远端服务器 + x server
  1. 服务端安装ssh服务
  1. 开启ssh 免密登陆, 具体可参考 开启vscode ssh 免密登陆章节。
  2. 安装 x client
  1. 运行xclock, 如果此时没有报错, 就说明x client安装好了, 可能这时候还没有ui, 因为x server还没准备好。

sudo apt-get install libx11-dev libxext-dev libxtst-dev libxrender-dev libxmu-dev libxmuu-dev # 相关的库

  1. 安装本地x server
    x server有两种, 一种是xming, 一种是VcXsrv, 前者收费而且年久失修, 后者是微软的, 这里选择后者。下载 传送门。安装比较简单, 直接next就可以。注意几点:

    • Display settings页面Display number设为-1, 选择多窗口
    • Client startup选择start no client

  2. 如果是vscode远程还需要安装一些扩展。

    • 服务端: Remote X11
    • 本地: Remote X11(SSH)
