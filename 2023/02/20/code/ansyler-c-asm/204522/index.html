简介

  某些场合需要对一些关键代码进行优化已满足高效的业务需求，因此，对于一些提供代码效率的优化研究就非常有必要，本文就基于汇编角度对一些常规代码用法进行效率比对和优化。

局部变量指针对齐效率影响

  我们知道, 代码运行过程中有堆区和栈区, 堆区通常用于Malloc动态内存分配空间，而栈区用于系统函数的调用(保存上级函数的调用现场)。既然是调用现场，那上级函数内的局部变量也就保存在栈空间上。这也就说明为什么在函数内不能够声明较大数组的原因(容易爆栈溢出)。

非数据结构多字节访问

  当访问的数据不是一个数据结构而是一个基础多字节数据类型(如int), 会出现两种情况:

  • 支持非对齐数据访问的硬件: 分为多次数据总线访问
  • 不支持非对齐数据访问的硬件: 会尝试直接进行非对齐数据访问(触发硬件异常)

结论:

  • 不对齐的结构体访问将导致效率下降, 变成单字节访问或者半字访问
  • 对于不支持非对齐地址访问的硬件，在处理结构体数据的时候即使地址不对齐也可以逐个数据访问。不会引发异常。
  • 大部分情况下非对齐数据访问效率要高于紧凑型数据访问。
