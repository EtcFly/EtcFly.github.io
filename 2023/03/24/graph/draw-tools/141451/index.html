  关键字node、edge、graph、digraph、subgraph是大小写无关的, 但是ID命名有如下要求:

  • 不以数字, 下划线开头
  • 任何通过双引号括起来的字符串
  • HTML字符串(<…>)

ID 只是一个字符串, 且abc_2和"abc_2" 之间没有语义差异。请注意，在 HTML 字符串中，尖括号必须成对出现，并且允许换行符和其他格式化空白字符。此外，内容必须是合法的 XML, 该语言支持 C++ 风格的注释：/* */ 和 // . 此外，以 # 字符开头的行被认为是从 C 预处理器输出的行（例如，#34 表示第 34 行）并被丢弃。

子图(subgraph)和集合(clusters)

子图有三个作用

  • 将 节点 和 边 组合到一起为一个整体
  • 子图可以提供设置属性的上下文
  • 如果子图的名称以 开头 cluster，则 Graphviz 将子图记录为特殊的集群子图

语义注释

  必须将图指定为(有向图)digraph 或 (无向图)graph, 从语义上讲，其区别在于从边的一个节点到另一个节点是否存在自然方向, 有向图以 -> 连接, 无向图以-连接。，默认情况下，绘制有向图中的边带有指向头节点的箭头。对于普通图，默认情况下绘制的边没有任何箭头。

rank

设置子图的节点等级约束

  • 如果rank="same"，则所有节点都放置在同一等级上。
  • 如果rank="min"，则所有节点都放在最小等级上。
  • 如果rank="source"，则所有节点都放在最小秩上，并且只有最小秩上的节点属于某个子图 rank="source"或rank="min"。

rankdir

  • TB: Top to bottom
  • BT: Bottom to top
  • LR: Left to right
  • RL: Right to left
