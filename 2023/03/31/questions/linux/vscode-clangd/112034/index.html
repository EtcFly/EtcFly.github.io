vscode配合clangd插件实现代码补全

原文地址：https://etcfly.top

背景

  上一章节已经介绍过基于 GNU Global 插件来实现代码补全和提示, GNU Global通过gtags工具首先生成对应文件的符号信息, 每次编码通过全局匹配来搜索相关补全信息, 这样做虽然速度较快, 但是有时候无法准确跳转和定位。哪有什么工具可以更加精准的匹配和补全吗？于是 clangd诞生了。

  Clangd是llvm项目推出的C++语言服务器，通过LSP(Language Server Protocal)协议向编辑器如vscode/vim/emacs提供语法补全、错误检测、跳转、格式化等等功能。其次, 出了这些基础功能, Clangd 还提供了一些特色功能:

  • 自动插入头文件

clangd的代码补全可以搜索你的整个代码库，即使是没有被include进来的变量也会覆盖到。而且在补全时，它会自动帮你填上合适的namespace和头文件

  • clang-format 格式化

clangd可以提供自动的clang-format格式化， 保证代码的格式统一。

  • 类型提示

从clangd-14开始，clangd提供变量自动的类型提示

  • Clang-tidy代码检查

对于一些潜在问题进行检查并给出提示

  • 内存占用率低

占用系统资源确实比 C/C++ 少了很多，无论是 CPU 还是 内存的使用

任何事情都有两面性, 同样对于clangd, 其缺点主要表现在:

  • 配置相较于 C/C++ 较繁琐。
  • 需要依赖第三方工具生成compile_commands.json

总结一下，就是对于大型项目来说，使用 clangd 能够明显降低系统资源的占用，能够减少出现系统卡顿的情况，不过使用门槛较高。C/C++插件 更适合小项目的开发，使用起来更加便利，能够更好的进行调试。
