异常模型

即 Python 程序的语法是正确的，在运行它的时候，也有可能发生错误。运行期检测到的错误被称为异常。

异常分为两类:

  • ASSART 主动抛出的异常
  • Except 系统抛出的异常

Note:

在异常try代码块只要return 0则产生异常

如下
就是一个系统抛出的异常, 当程序里出现除0异常时, 系统自动抛出异常。

执行流程:

  • 首先执行检测代码, 代码发生异常, 检测代码余下部分被忽略, 跳到异常处理代码
  • 代码异常执行 except, 正常时不执行, [error code] 为异常代码, 可以是单个匹配, 也可以是元组, 多个异常公用一个处理程序, 如果忽律不写, 那么默认处理所有异常。
    • 如果一个异常没有与任何的 except 匹配，那么这个异常将会传递给上层的 try 中。
    • 一个 try 语句可能包含多个except子句，分别来处理不同的特定的异常。最多只有一个分支会被执行
    • 一个 except 子句可以同时处理多个异常，这些异常将被放在一个括号里成为一个元组
  • 代码正常时执行, 抛出异常时不执行
  • finally 无论是否异常都会执行
